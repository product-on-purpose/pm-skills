# Foundation Sprint Skill: Sample Output Artifacts

**Purpose:** These are spec-validation drafts. Each sample shows the bundled output a Foundation Sprint skill would produce when invoked. The goal is to concretize each spec's "Bundled output" section, surface gaps in the design, and inform the eventual SKILL.md / TEMPLATE.md / EXAMPLE.md authoring during v2.15.0 Phase 3.

**Not the v2.15.0 library samples.** The public-facing library samples (3 threads x 8 FS files) ship in Phase 5 at `library/skill-output-samples/tool-foundation-sprint-*/`. Those are authored when the skills are shipping. These internal samples exist to validate the design before that.

## Thread: Brainshelf book-catalog

All samples share the Brainshelf narrative for cross-skill coherence. Brainshelf is a fictional pre-seed B2C SaaS for personal book collection management.

| Element | Value |
|---|---|
| Company | Brainshelf (fictional) |
| Stage | Pre-seed; pre-product |
| Strategic question | Build for individual collectors (private library) or social readers (community)? |
| Decider | Jamie (founder, PM) |
| Sprint team | Jamie (Decider, PM), Alex (design), Sam (engineering), Riley (customer expert from social-reading community) |
| Sprint dates | 2026-05-13 to 2026-05-14 |

The Founding Hypothesis the sprint produces (the destination, not a constraint on the path): the team converges on personal-collection enthusiasts who want low-friction private library tracking, betting against the social/community direction.
