# Design Sprint Skill: Sample Output Artifacts

**Purpose:** These are spec-validation drafts. Each sample shows the bundled output a Design Sprint skill would produce when invoked. The goal is to concretize each spec's "Bundled output" section, surface gaps in the design, and inform the eventual SKILL.md / TEMPLATE.md / EXAMPLE.md authoring during v2.15.0 Phase 2 of the Design Sprint integration plan.

**Not the v2.15.0 library samples.** The public-facing library samples (3 threads x 7 DS files = 21 files) ship in Phase 5 at `library/skill-output-samples/tool-design-sprint-*/`. Those are authored when the skills are shipping. These internal samples exist to validate the design before that.

## Thread: Brainshelf book-catalog (continuation)

These samples continue the Brainshelf narrative from the Foundation Sprint samples. The Brainshelf team completed their Foundation Sprint and produced a Founding Hypothesis around personal-collection enthusiasts. The Design Sprint now tests that hypothesis with target customers.

| Element | Value |
|---|---|
| Company | Brainshelf (fictional) |
| Stage | Post-Foundation-Sprint; pre-product |
| Sprint challenge | Will personal-collection enthusiasts adopt low-friction private library tracking over Goodreads / paper / nothing? |
| Decider | Jamie (founder, PM) |
| Sprint team | Jamie (Decider, PM), Alex (design), Sam (engineering), Riley (customer expert, recruiter) |
| Sprint dates | Monday 2026-05-26 through Friday 2026-05-30 |
| Customer participants (Friday) | 5 target customers recruited from Riley's book-blogger network |

The handoff between sprints is narrative (no bridge skill); the Founding Hypothesis output feeds into Day 1 strategic context. See `_workflows/foundation-to-design.md` for the canonical handoff documentation.
