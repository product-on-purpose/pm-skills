# PM-Skills

> 40 plug-and-play product management skills your AI agent can run today. PRDs, OKRs, hypotheses, retros, opportunity trees, and 35 more, each with a template, a worked example, and a slash command.

[![Version](https://img.shields.io/badge/version-2.13.1-blue?style=flat-square)](https://github.com/product-on-purpose/pm-skills/releases/latest) [![Skills](https://img.shields.io/badge/skills-40-brightgreen?style=flat-square)](#skills-at-a-glance) [![Spec](https://img.shields.io/badge/spec-agentskills.io-orange?style=flat-square)](https://agentskills.io/specification) [![License](https://img.shields.io/badge/license-Apache_2.0-blue?style=flat-square)](LICENSE)

## Try it in 60 seconds

```bash
npx skills add product-on-purpose/pm-skills
```

That's it. Forty PM skills now load into your agent. Try one:

```
/prd "A focus-mode feature for our task app"
```

The agent reads `skills/deliver-prd/SKILL.md`, follows the template at `references/TEMPLATE.md`, mirrors the worked example at `references/EXAMPLE.md`, and produces a structured PRD with problem statement, success metrics, user stories, scope, and technical considerations. No prompt engineering. No starting from zero.

Works in Claude Code, Cursor, GitHub Copilot, Cline, and any agent supported by the open [`skills` CLI](https://github.com/vercel-labs/skills).

## What you get

Every skill is a directory with three files plus a slash command:

```
skills/deliver-prd/
├── SKILL.md                      <- agent instructions
├── references/
│   ├── TEMPLATE.md               <- output shape
│   └── EXAMPLE.md                <- worked example, ~200-400 lines
commands/prd.md                   <- /prd slash command
```

Browse [`library/skill-output-samples/`](library/skill-output-samples/) for 126 real outputs across three fictional product narratives (Storevine: B2B ecommerce; Brainshelf: consumer PKM; Workbench: enterprise collaboration). Every sample shows the prompt the PM typed, the artifact the skill produced, and source notes citing real public references.

## Why this exists

Every prompt to an AI for product management work starts from zero by default. Generic responses, inconsistent formats, missing critical sections. PM-Skills replaces "explain to the AI what a good PRD looks like" with "the AI already knows, because it's reading a battle-tested template."

The 40 skills are based on the [Triple Diamond framework](https://medium.com/zendesk-creative-blog/the-zendesk-triple-diamond-process-fd857a11c179) (six product phases extending the Design Council's Double Diamond), [Teresa Torres' opportunity solution trees](https://www.producttalk.org/opportunity-solution-tree/), [Jobs to be Done](https://jtbd.info/), [Architecture Decision Records (Michael Nygard format)](https://adr.github.io/), and the [Agent Skills Specification](https://agentskills.io/specification).

Three skills (`/pm-skill-builder`, `/pm-skill-validate`, `/pm-skill-iterate`) form a Create > Validate > Iterate loop for evolving the library itself. Skills are living artifacts; the lifecycle tools keep the library consistent as it grows.

## FAQ

**Does this work without Claude Code?**
Yes. Cursor, Copilot, Windsurf, Claude.ai, Claude Desktop, OpenCode, any MCP client, and any agent that reads `AGENTS.md` or supports the `skills` CLI. Slash-command UX is best in Claude Code, but the markdown skills work everywhere.

**Can I customize skills for my team?**
Yes. Fork, edit `skills/{phase-skill}/SKILL.md` directly, change the template, change the example. Apache 2.0 license covers commercial use.

**How do I add a new skill?**
Run `/pm-skill-builder "describe your skill"`. It runs gap analysis against existing skills, classifies the new one, and generates draft files for review. Then `/pm-skill-validate` checks repo conventions before merge.

**What's the difference between this repo and pm-skills-mcp?**
This repo is the live skill library (markdown files, active maintenance). [pm-skills-mcp](https://github.com/product-on-purpose/pm-skills-mcp) is an MCP server wrapping a frozen v2.9.2 build of the same catalog (maintenance mode, security patches only). Use this repo for slash commands and customization; use MCP if your tooling specifically requires the protocol.

**Where can I see the skills in action?**
[`library/skill-output-samples/`](library/skill-output-samples/) has 126 worked examples organized as three product narratives. Each sample shows the prompt + the output + source notes.

## Contributing

PRs welcome. Quickstart for contributors:

1. Fork, branch.
2. Make changes (or run `/pm-skill-builder` for new skills).
3. Run `bash scripts/lint-skills-frontmatter.sh` and `bash scripts/validate-plugin-install.sh` locally.
4. Open a PR.
