## What it is

Every time you ask an AI to help with product management, you start from zero. PM-Skills changes that. It is a curated library of 40 best-practice skills, each one a specialized capability your AI agent can invoke on demand: write a PRD, draft hypotheses, score OKRs, frame a problem, build an opportunity tree, run a retro. Each skill ships with a template, a worked example, and a slash command.

| Without PM-Skills | With PM-Skills |
|---|---|
| Generic AI responses | Battle-tested PM frameworks |
| Inconsistent formats | Production-ready templates |
| Missing key sections | Comprehensive coverage |
| Prompt-engineering every time | One command, instant output |

Compatible with Claude Code, Claude.ai, Claude Desktop, GitHub Copilot, Cursor, Windsurf, VS Code, and any agent supported by the open `skills` CLI. Optional MCP server (`pm-skills-mcp`) for protocol-level access.

## Getting Started

Pick the install path that matches your tool. Most users want option 1 or 2.

| # | Path | Command | Best for |
|---|---|---|---|
| 1 | **`skills` CLI** | `npx skills add product-on-purpose/pm-skills` | Most agents (Claude Code, Cursor, Copilot, Cline). Fastest path. |
| 2 | **Claude Code plugin** | `/plugin marketplace add product-on-purpose/pm-skills` then `/plugin install pm-skills@pm-skills-marketplace` | Modern Claude Code with the plugin marketplace |
| 3 | **Git clone** | `git clone https://github.com/product-on-purpose/pm-skills.git` | Forkers, contributors, AGENTS.md auto-discovery (Copilot, Cursor, Windsurf) |
| 4 | **ZIP download** | [Latest release](https://github.com/product-on-purpose/pm-skills/releases/latest) | Claude.ai, Claude Desktop, offline install |
| 5 | **MCP server** | `npx pm-skills-mcp` | MCP clients. Maintenance mode; v2.9.x line. |

After install, slash commands like `/prd`, `/hypothesis`, `/opportunity-tree`, `/okr-writer`, `/agenda` resolve immediately.

```
/prd "Search feature for our e-commerce platform"
/hypothesis "One-page checkout will lift conversion"
/workflow-feature-kickoff "Recurring tasks"
```

For the full per-platform setup walkthrough, see [docs/getting-started](docs/getting-started/index.md).

## Skills

40 skills across 8 categories. Each ships with a `SKILL.md` (instructions), a `references/TEMPLATE.md` (output shape), a `references/EXAMPLE.md` (worked example), and a slash command.

| Phase / category | Skills | Slash commands |
|---|---|---|
| **Foundation** (cross-cutting) | persona, lean-canvas, okr-writer, meeting-agenda, meeting-brief, meeting-recap, meeting-synthesize, stakeholder-update | `/persona`, `/lean-canvas`, `/okr-writer`, `/agenda`, `/meeting-brief`, `/meeting-recap`, `/meeting-synthesize`, `/stakeholder-update` |
| **Discover** (find the right problem) | competitive-analysis, interview-synthesis, stakeholder-summary | `/competitive-analysis`, `/interview-synthesis`, `/stakeholder-summary` |
| **Define** (frame the problem) | problem-statement, hypothesis, opportunity-tree, jtbd-canvas | `/problem-statement`, `/hypothesis`, `/opportunity-tree`, `/jtbd-canvas` |
| **Develop** (explore solutions) | solution-brief, spike-summary, adr, design-rationale | `/solution-brief`, `/spike-summary`, `/adr`, `/design-rationale` |
| **Deliver** (ship it) | prd, user-stories, edge-cases, acceptance-criteria, launch-checklist, release-notes | `/prd`, `/user-stories`, `/edge-cases`, `/acceptance-criteria`, `/launch-checklist`, `/release-notes` |
| **Measure** (validate with data) | experiment-design, instrumentation-spec, dashboard-requirements, experiment-results, okr-grader | `/experiment-design`, `/instrumentation-spec`, `/dashboard-requirements`, `/experiment-results`, `/okr-grader` |
| **Iterate** (learn and improve) | retrospective, lessons-log, refinement-notes, pivot-decision | `/retrospective`, `/lessons-log`, `/refinement-notes`, `/pivot-decision` |
| **Utility** (meta-tooling) | pm-skill-builder, pm-skill-validate, pm-skill-iterate, mermaid-diagrams, slideshow-creator, update-pm-skills | `/pm-skill-builder`, `/pm-skill-validate`, `/pm-skill-iterate`, `/mermaid-diagrams`, `/slideshow-creator`, `/update-pm-skills` |

Skills follow the [Triple Diamond framework](https://medium.com/zendesk-creative-blog/the-zendesk-triple-diamond-process-fd857a11c179). Each one is a self-contained markdown file you can read, fork, and adapt.

### Skill lifecycle tools

Three utility skills form a Create > Validate > Iterate loop for the library itself:

- **`/pm-skill-builder`**: gap analysis, classification, generates draft files.
- **`/pm-skill-validate`**: audits a skill against repo conventions, produces a severity-graded report.
- **`/pm-skill-iterate`**: applies fixes from feedback or a validation report, suggests a version bump.

See [PM-Skill Lifecycle](docs/guides/pm-skill-lifecycle.md) for the full pattern.

## pm-skills vs pm-skills-mcp

|  | pm-skills (this repo) | [pm-skills-mcp](https://github.com/product-on-purpose/pm-skills-mcp) |
|---|---|---|
| Format | Markdown skill library | MCP server wrapping the library |
| Setup | `npx skills add ...` or git clone, 2-5 min | `npx pm-skills-mcp`, 30 sec |
| Invocation | Slash commands or AGENTS.md | MCP tool calls |
| Updates | `git pull` (active) | Maintenance mode (security + critical only) |
| Catalog | Live, all 40 skills | Frozen at v2.9.2 build (40 skills) |

Use **this repo** for slash commands, customization, and forking. Use **MCP** if your tooling specifically requires the protocol.

## FAQ

**Do I need Claude Code?** No. PM-Skills works with Cursor, Copilot, Windsurf, OpenCode, Claude.ai, Claude Desktop, and any MCP client. Slash-command UX is best in Claude Code.

**Can I fork and customize skills?** Yes. The Apache 2.0 license covers commercial and personal use. The whole library is markdown; edit any `SKILL.md` to change behavior.

**How do I add a new skill?** Use `/pm-skill-builder` for the guided path, or hand-author per [docs/guides/creating-pm-skills.md](docs/guides/creating-pm-skills.md). Then `/pm-skill-validate` checks conventions.

**What does Triple Diamond mean?** Six product phases (Discover > Define > Develop > Deliver > Measure > Iterate) extending the Design Council's Double Diamond. Each phase has 3 to 6 skills. See [docs/concepts/triple-diamond-delivery-process.md](docs/concepts/triple-diamond-delivery-process.md).

**Where can I see the skills in action?** [`library/skill-output-samples/`](library/skill-output-samples/) has 126 worked examples across three fictional product narratives (Storevine, Brainshelf, Workbench). Browse the [README_SAMPLES.md](library/skill-output-samples/README_SAMPLES.md) index.
