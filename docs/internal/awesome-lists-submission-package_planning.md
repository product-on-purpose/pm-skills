## Workflow for Each Submission

1. **Research** (before creating issue)
   - Review target repository structure
   - Read contribution guidelines
   - Check for existing pm-skills entry
   - Confirm fit with repository scope
   - Review existing entries for format/style

2. **Create GitHub Issue**
   - Copy content from issue draft file
   - Create issue in product-on-purpose/pm-skills
   - Add issue number to this planning doc

3. **Prepare Submission**
   - Fork target repository
   - Create branch for submission
   - Add pm-skills entry following their format
   - Use positioning from issue draft

4. **Submit PR**
   - Submit PR with tailored description
   - Link PR in GitHub issue
   - Link PR in this planning doc

5. **Follow-up**
   - Respond to maintainer feedback
   - Make requested changes
   - Update issue and planning doc with status

6. **Close Task**
   - Update result in this planning doc
   - Add learnings/notes
   - Close GitHub issue with reference to this doc
   - Update summary statistics
