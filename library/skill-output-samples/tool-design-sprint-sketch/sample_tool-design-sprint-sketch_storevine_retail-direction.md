## Sketch Assignment Plan

**Approach:** Swarm. All 4 sketch the first-30-sec scan moment.

**Format:** Each sketch shows the email-open view + the first PDF/web page view side-by-side.

**Time:** Notes 20 min / Ideas 20 min / Crazy 8s 8 min / Solution Sketch 75 min.

## Four-Step Sketches

### Sketch A (Carlos's; attribution stripped Wednesday morning)

**Solution Sketch description:** Email subject "Monday: 3 actions for [Store Name] - Carlos's read". Email body opens with a single sentence: "This week's top action is restock [SKU 4421] at [Store 12] before Saturday." Below: "See 2 more actions" link. PDF attachment shows top-3 as numbered cards with SKU + qty + store + 1-line rationale. Carlos's name + photo + "Reviewed by Carlos, ex-REI buyer" sidebar.

**Distinctive elements:** Named-analyst attribution as primary trust signal; subject line carries top action; first sentence is the action, not the framing.

### Sketch B (Tasha's; attribution stripped)

**Solution Sketch description:** Email subject "Monday Brief - Week of July 13". Email body is a visual card-stack preview (3 mini-cards visible). Tap a card to open the web companion. PDF attachment is the printable version of the same 3 cards. First card above-fold; below-fold contains data visualization + analyst reasoning.

**Distinctive elements:** Card-stack visual metaphor; mobile-first; web companion = primary, PDF = secondary print artifact.

### Sketch C (Devon's; attribution stripped)

**Solution Sketch description:** Email subject includes severity badge ("HIGH-IMPACT WEEK"). Body is a structured prioritization list (Top 3 actions with priority badges + dollar-impact projection). PDF is essentially the email content as a 1-page print. No companion web page.

**Distinctive elements:** Quantified impact per action ($-value projection); severity color-coding; PDF-only (rejects fragmentation).

### Sketch D (Mei's; attribution stripped)

**Solution Sketch description:** Email body opens with "What changed this week" 1-paragraph narrative followed by "What to do about it" 3-bullet action list. Action bullets are specific (SKU + store + qty). PDF mirrors email + adds deeper analyst commentary section. Web companion exists only as drill-down per action.

**Distinctive elements:** Narrative-led ("what changed") before action list; clear PDF/web split (PDF = read; web = drill-down).
